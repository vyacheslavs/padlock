протокол

1. клиент присоединяется по пути /api/v1/action/ehlo, чтобы выяснить какой сервер будет обслуживать его

2. клиент получает json ввиде: {padlock_server: {host: "keepass2.ldd.so", port: 8008}, version: 1}
   это значит, что клиента будет обслуживать keepass2.ldd.so сервер на порту 8008

3. клиент присоединяется к новому адресу по пути /api/v1/action/helo (не ehlo) и должен получить json:
   {status: "online", version: 1}
   статус online означает, что сервер готов обслуживать этого клиента
